# 学习路线：从能跑到会调

这份路线解决一个问题：目录里文件很多，应该先看什么、后看什么、每篇文档到底服务于导航系统的哪一部分。

## 1. 先不要从理论开始

学习 Nav2 最容易卡住的地方，是一上来就读行为树、代价地图、TF、插件，结果每个词都认识，但不知道它们什么时候出现。

更好的顺序是：

1. 先跑通一个完整导航例子。
2. 再回头解释这个例子里每个节点和参数为什么存在。
3. 最后专门练调参和故障诊断。

## 2. 第一条主线：最小导航闭环

先读 `实践操作/Turtlebot3仿真导航快速跑通.md`。

这篇文档只回答一个问题：**一个标准仿真导航任务最少需要哪些步骤？**

你会看到：

1. Gazebo 里出现机器人和场景。
2. Nav2 启动地图、定位、规划、控制、RViz。
3. RViz 里先用 `2D Pose Estimate` 对齐定位。
4. RViz 里用 `Nav2 Goal` 下发终点。
5. 机器人自动规划并到达终点。

这是后面所有知识的参照物。后面每个文档都应该能对应到这里的一步。

## 3. 第二条主线：机器人为什么能被仿真出来

读：

- `实践操作/机器人模型URDF与RViz可视化.md`
- `实践操作/Gazebo仿真与EKF里程计.md`

它们回答：

1. 机器人长什么样？由 URDF/Xacro 描述。
2. 机器人各部件坐标在哪里？由 joint 和 TF 表达。
3. 机器人怎么在 Gazebo 里动？由差速驱动插件模拟。
4. 机器人怎么产生 `/scan`、`/odom`、`/imu`？由 Gazebo 传感器插件模拟。
5. 机器人局部位姿怎么稳定？由 EKF 融合里程计和 IMU。

如果机器人“不动”或“RViz 看不到模型/雷达”，先回到这两篇。

## 4. 第三条主线：机器人怎么知道自己在哪

读：

- `实践操作/传感器建图定位与代价地图.md`
- `实践操作/SLAM边建图边导航.md`
- `理论知识/Nav2状态估计与TF.md`

它们回答：

1. `/scan` 是怎么变成地图的？
2. SLAM 和 AMCL 有什么区别？
3. `map -> odom -> base_link` 这条 TF 链为什么必须存在？
4. 为什么仿真中所有节点都要 `use_sim_time:=True`？

判断标准：

```bash
ros2 run tf2_ros tf2_echo map odom
ros2 run tf2_ros tf2_echo odom base_link
ros2 topic echo /scan --once
ros2 topic echo /map --once
```

如果这些不通，先别调路径参数，导航大脑还没拿到可靠世界观。

## 5. 第四条主线：机器人怎么判断哪里危险

读：

- `实践操作/Nav2代价地图参数配置.md`
- `实践操作/机器人Footprint轮廓配置.md`
- `理论知识/Nav2环境表达与代价地图.md`

它们回答：

1. 为什么普通黑白地图不能直接导航？
2. Global Costmap 和 Local Costmap 分别管什么？
3. Static Layer、Obstacle Layer、Voxel Layer、Inflation Layer 分别做什么？
4. `robot_radius` 和 `footprint` 为什么会影响过门、贴墙、碰撞？

调参时优先看：

```yaml
robot_radius
footprint
inflation_radius
cost_scaling_factor
obstacle_max_range
raytrace_max_range
width
height
update_frequency
```

## 6. 第五条主线：机器人怎么规划和执行路径

读：

- `理论知识/Nav2通信与生命周期.md`
- `理论知识/Nav2行为树与导航服务器.md`
- `实践操作/Nav2规划器与控制器插件配置.md`

它们回答：

1. Planner 负责“路线怎么走”。
2. Controller 负责“车现在怎么动”。
3. Recovery 负责“卡住以后怎么自救”。
4. 行为树负责调度这些服务器。

如果现象是“能生成路径但走得不好”，读这部分。

## 7. 第六条主线：特殊任务

普通到点导航稳定以后，再读：

- `进阶专题/行为树动态目标追踪.md`
- `进阶专题/导航禁止区KeepoutZones配置.md`
- `进阶专题/Groot行为树可视化编辑.md`
- `进阶专题/Nav2自定义行为树插件编写.md`
- `理论知识/行为树黑板机制.md`
- `进阶专题/STVL外部代价地图插件.md`

这些不是入门必需，而是特殊题型工具箱。

## 8. 最终学习目标

读完这套资料，不是要背下所有参数，而是要能做到：

1. 看到题目，判断是已知地图导航还是 SLAM 导航。
2. 找到启动文件和 `nav2_params.yaml`。
3. 启动 Gazebo、RViz、SLAM/AMCL、Nav2。
4. 用 TF、topic、lifecycle 快速判断系统是否完整。
5. 根据机器人表现修改 costmap、footprint、planner、controller 参数。
6. 成功到点后截图提交。

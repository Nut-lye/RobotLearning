# Nav2 导航资料入口

这个目录是一套面向 ROS 2 Humble / Nav2 / Gazebo 仿真导航的学习与实操资料。它不是按“知识点堆叠”来读的，而应该按“先跑通、再理解、最后调参”的顺序使用。

如果你是第一次打开这个目录，先读：

1. `00_学习路线.md`：告诉你每个文件在整个导航系统中负责什么。
2. `模块C知识源整理.md`：把比赛/训练任务翻译成 Nav2 工程动作。
3. `实践操作/目的对应操作指南.md`：逐条对应 `目的.md` 的要求，说明要怎么操作。
4. `实践操作/模块C机器人导航教学实现.md`：按技术文件模块 C，用 `src/module_c_nav_teaching` 课程包分场景练习。
5. `实践操作/Turtlebot3仿真导航快速跑通.md`：先跑通一个官方最小导航闭环。
6. `实践操作/Nav2调参与故障诊断手册.md`：机器人能动以后，用它定位问题和改参数。
7. `实践操作/实践提交与截图清单.md`：最后按清单检查截图和答题报告材料。

## 推荐阅读顺序

### 第一阶段：先建立导航闭环

目标：知道“能导航”到底长什么样。

- `实践操作/Turtlebot3仿真导航快速跑通.md`
- `实践操作/SLAM边建图边导航.md`

读完应该能做到：

- 启动 Gazebo。
- 启动 Nav2 和 RViz。
- 用 `2D Pose Estimate` 初始化定位。
- 用 `Nav2 Goal` 下发目标点。
- 看懂机器人为什么能从起点到终点。

### 第二阶段：理解导航系统的底座

目标：知道机器人为什么需要模型、仿真、传感器、TF 和定位。

- `实践操作/机器人模型URDF与RViz可视化.md`
- `实践操作/Gazebo仿真与EKF里程计.md`
- `实践操作/传感器建图定位与代价地图.md`
- `理论知识/Nav2理论总入口.md`
- `理论知识/Nav2通信与生命周期.md`
- `理论知识/Nav2行为树与导航服务器.md`
- `理论知识/Nav2状态估计与TF.md`
- `理论知识/Nav2环境表达与代价地图.md`

读完应该能解释：

- URDF 描述机器人身体。
- Gazebo 模拟底盘、雷达、IMU。
- TF 串起 `map -> odom -> base_link -> sensor`。
- SLAM 或 AMCL 负责 `map -> odom`。
- EKF 或底盘里程计负责 `odom -> base_link`。

### 第三阶段：掌握导航调参

目标：让机器人走得稳、不过度贴墙、不乱绕、不转圈。

- `实践操作/Nav2代价地图参数配置.md`
- `实践操作/机器人Footprint轮廓配置.md`
- `实践操作/Nav2规划器与控制器插件配置.md`
- `实践操作/Nav2调参与故障诊断手册.md`

读完应该能处理：

- 路径贴墙。
- 不敢过门。
- 局部避障失败。
- 到点附近转圈。
- 能规划但不执行。

### 第四阶段：特殊题型和进阶能力

目标：处理不是“普通到点导航”的任务。

- `进阶专题/行为树动态目标追踪.md`
- `进阶专题/导航禁止区KeepoutZones配置.md`
- `进阶专题/Groot行为树可视化编辑.md`
- `进阶专题/Nav2自定义行为树插件编写.md`
- `理论知识/行为树黑板机制.md`
- `进阶专题/STVL外部代价地图插件.md`

读完应该知道：

- 动态目标追踪靠行为树重构。
- 禁止区靠 keepout mask 和 costmap filter。
- Groot 主要用于离线编辑行为树。
- 自定义 BT 插件用于扩展导航行为。
- 外部代价插件用于更复杂的 3D 感知避障。

## 一句话总览

完整导航链路是：

```text
URDF 机器人模型
  -> Gazebo 生成机器人和传感器数据
  -> TF / odom / scan / map 数据齐全
  -> SLAM 或 AMCL 完成定位
  -> Costmap 把地图和障碍物变成风险网格
  -> Planner 规划全局路径
  -> Controller 输出 cmd_vel 控制机器人
  -> RViz 下发目标并观察结果
```

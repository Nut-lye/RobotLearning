# ROS 2 导航核心原理解析：建图、定位与环境感知

在移动机器人导航系统中，硬件传感器（如激光雷达和深度相机）只能提供原始的物理测量数据（距离或点云）。为了实现真正的“自主导航”，机器人必须具备**构建环境认知（建图）**、明确自身位置（定位）**以及**评估行驶安全性（代价地图）的能力。

## 1. 机器人的世界观与自我认知：建图与定位

在 Nav2 架构中，建图（Mapping）与定位（Localization）是紧密耦合的两个基础模块。它们共同解决了一个核心问题：计算并发布 `map` $\rightarrow$ `odom` 的坐标系转换（TF）。

### 1.1 SLAM 建图 (Simultaneous Localization and Mapping)

- **理论核心：** 当机器人处于一个完全陌生的环境中时，它需要在移动的同时，利用激光雷达（`LaserScan`）不断探测周围的障碍物边界，拼接出一张全局的静态二维地图（Occupancy Grid Map）。
- **Nav2 实现 (`slam_toolbox`)：** Nav2 官方推荐使用 `slam_toolbox`。该算法在建图的同时，会反向推算机器人当前的位姿。它通过对比“当前雷达扫描到的特征”与“已经建好的地图轮廓”，计算出里程计由于打滑而产生的误差，并持续发布 `map` $\rightarrow$ `odom` 的修正 TF。

### 1.2 AMCL 蒙特卡洛定位

- **理论核心：** 当机器人已经拥有了一张提前建好的静态地图时，它不再需要 SLAM。此时它使用自适应蒙特卡洛定位（AMCL）算法。AMCL 基于粒子滤波理论，在地图上撒布大量代表可能位置的“粒子”。随着机器人移动和雷达扫描，与实际环境匹配度高的粒子被保留，匹配度低的被淘汰，最终粒子群收敛于机器人的真实坐标。
- **Nav2 实现 (`nav2_amcl`)：** 该节点接管了 SLAM 的定位工作，同样负责发布稳健的 `map` $\rightarrow$ `odom` 转换，确保机器人在已知地图中的绝对位置精度。

## 2. 机器人的安全边界：环境表达与代价地图 (Costmap 2D)

原始的传感器数据和黑白静态地图无法直接供导航算法（A* 或 DWA）使用。Nav2 引入了**代价地图（Costmap 2D）**，将物理空间转化为带有“成本”数值的网格图。

### 2.1 代价量化模型

在代价地图中，空间被划分为一个个极小的网格（Cell，通常为 5cm×5cm）。每个网格被赋予 0 到 254 的“代价值（Cost）”：

- **0（Free Space）：** 绝对安全的自由空间。
- **1 ~ 252（Non-lethal Obstacle）：** 缓冲区域。数值越高代表越靠近障碍物，规划器在寻路时会尽量避开高代价区域。
- **253（Inscribed Inflated Obstacle）：** 内切障碍物区。代表机器人的边缘已经碰到障碍物。
- **254（Lethal Obstacle）：** 致命障碍物区。代表物理障碍物的绝对中心。

### 2.2 图层叠加架构 (Layered Architecture)

真实环境是复杂多变的，Nav2 的 `nav2_costmap_2d` 包采用了高度解耦的**多图层叠加机制**，从下至上合成最终的代价地图：

1. **静态层 (Static Layer)：**
   - 直接订阅建图模块（如 `slam_toolbox`）发布的静态地图数据（`/map`）。
   - 它构成了代价地图的底色，代表了不可改变的墙壁和固定设施。
2. **障碍物层 / 体素层 (Obstacle / Voxel Layer)：**
   - **动态感知核心。** 实时订阅雷达的 `/scan` 或相机的 `/pointcloud2` 话题。
   - 当环境中出现未记录在静态地图上的动态物体（如行人、临时放置的纸箱）时，这一层会实时在地图上打上 254 的高代价标记；当物体移走时，通过光线追踪（Raytrace）技术实时清除标记。体素层还能处理 3D 悬空障碍物。
3. **膨胀层 (Inflation Layer)：**
   - **安全防碰撞核心。** 机器人不是一个没有体积的质点。膨胀层会以所有 254 致命障碍物为中心，根据机器人的几何半径（`robot_radius`）向外进行代价梯度的扩散。
   - 这保证了规划器计算出的路径永远与墙壁保持至少一个“机器人半径”的安全距离。

### 2.3 宏观与微观的解耦：全局与局部代价地图

为了兼顾“长途规划的效率”与“紧急避障的实时性”，系统并行维护两张代价地图：

- **全局代价地图 (Global Costmap)：**
  - **参考系：** `map` 坐标系。
  - **覆盖范围：** 整个已知环境。
  - **用途：** 运行全局路径规划算法（如 A*、Dijkstra），为机器人计算一条从当前位置到几十米外目标点的宏观最优路线。更新频率较低（通常 1Hz）。
- **局部代价地图 (Local Costmap)：**
  - **参考系：** `odom` 坐标系。
  - **覆盖范围：** 以机器人为中心的滑动小窗口（如 3m × 3m）。
  - **用途：** 运行局部轨迹控制算法（如 DWB、TEB），专职负责紧前方的动态避障。它以极高的频率更新（通常 5Hz 以上），随时应对突然冲入视野的行人或障碍物。

### 第 3 步：覆盖完整的 Launch 启动脚本

 `launch/display.launch.py`，将下面这份包含了**状态发布 + 世界加载 + Gazebo启动 + 模型注入 + 卡尔曼滤波 + RViz** 的代码放进去：

```python
import launch
from launch.substitutions import Command, LaunchConfiguration
import launch_ros
import os

def generate_launch_description():
    pkg_share = launch_ros.substitutions.FindPackageShare(package='sam_bot_description').find('sam_bot_description')
    default_model_path = os.path.join(pkg_share, 'src/description/sam_bot_description.urdf')
    default_rviz_config_path = os.path.join(pkg_share, 'rviz/urdf_config.rviz')
    ekf_config_path = os.path.join(pkg_share, 'config/ekf.yaml')
    
    # 指向我们在第 1 步下载的世界文件
    world_path = os.path.join(pkg_share, 'world/my_world.sdf')

    robot_state_publisher_node = launch_ros.actions.Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        parameters=[{'robot_description': Command(['xacro ', LaunchConfiguration('model')])}]
    )

    # 启动 Gazebo 并加载世界
    start_gazebo_cmd = launch.actions.ExecuteProcess(
        cmd=['gazebo', '--verbose', '-s', 'libgazebo_ros_init.so', '-s', 'libgazebo_ros_factory.so', world_path], 
        output='screen'
    )

    # 把机器人扔进 Gazebo 世界
    spawn_entity = launch_ros.actions.Node(
        package='gazebo_ros',
        executable='spawn_entity.py',
        arguments=['-entity', 'sam_bot', '-topic', 'robot_description'],
        output='screen'
    )

    # 启动 EKF 卡尔曼滤波
    robot_localization_node = launch_ros.actions.Node(
       package='robot_localization',
       executable='ekf_node',
       name='ekf_filter_node',
       output='screen',
       parameters=[ekf_config_path, {'use_sim_time': True}]
    )

    rviz_node = launch_ros.actions.Node(
        package='rviz2',
        executable='rviz2',
        name='rviz2',
        output='screen',
        arguments=['-d', LaunchConfiguration('rvizconfig')],
        parameters=[{'use_sim_time': True}]
    )

    return launch.LaunchDescription([
        launch.actions.DeclareLaunchArgument(name='model', default_value=default_model_path),
        launch.actions.DeclareLaunchArgument(name='rvizconfig', default_value=default_rviz_config_path),
        robot_state_publisher_node,
        start_gazebo_cmd,
        spawn_entity,
        robot_localization_node,
        rviz_node
    ])

```

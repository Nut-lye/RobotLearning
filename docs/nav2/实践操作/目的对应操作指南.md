# 目的对应操作指南

这篇文档专门回答 `/home/nuts_2204/ros2_ws/doc/nav_doc/目的.md` 中的要求：这些目标在 `实践操作/` 目录里是否都有覆盖，以及实际应该怎么操作。

结论：**都有覆盖**。但它们分散在不同实践文档里，因此这里把目标、对应文档、操作步骤、验证标准集中列出来。

## 1. 目标一：加载预设导航场景

题目原意：

```text
打开给定的 Gazebo / RViz / ROS2 导航环境。
```

对应实践文档：

- `Turtlebot3仿真导航快速跑通.md`
- `SLAM边建图边导航.md`
- `机器人模型URDF与RViz可视化.md`
- `Gazebo仿真与EKF里程计.md`

### 1.1 如果使用 Turtlebot3 官方场景

本仓库推荐先使用本地 world 版本，避免 Gazebo 启动时去在线模型库下载资源：

```bash
cd ~/ros2_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch tb3_nav_practice turtlebot3_world_simtime0.launch.py
```

这个启动文件会设置：

```bash
GAZEBO_MODEL_PATH=/home/.../tb3_nav_practice/models:/opt/ros/humble/share/turtlebot3_gazebo/models:/usr/share/gazebo-11/models
GAZEBO_MODEL_DATABASE_URI=
```

如果只是跑官方例子，也可以用下面的命令。

终端 1：

```bash
source /opt/ros/humble/setup.bash
export TURTLEBOT3_MODEL=waffle
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
```

预期现象：

- Gazebo 打开。
- 世界中出现 Turtlebot3 机器人。
- 场景中有墙体、障碍物或测试房间。

### 1.2 如果使用自定义机器人包

先找到包名和 launch 文件：

```bash
cd ~/ros2_ws
find src -path '*launch*' -name '*.launch.py'
```

启动：

```bash
cd ~/ros2_ws
source install/setup.bash
ros2 launch <package_name> <launch_file>.launch.py
```

预期现象：

- Gazebo 或 RViz 启动。
- 机器人模型正常显示。
- 没有明显 TF 报错。

### 1.3 这一阶段要检查什么

```bash
ros2 topic list
ros2 topic echo /scan --once
ros2 topic echo /odom --once
ros2 run tf2_ros tf2_echo odom base_link
```

判断：

- 有 `/scan`：雷达工作。
- 有 `/odom`：里程计工作。
- 有 `odom -> base_link`：机器人位姿链路工作。

截图建议：

- Gazebo 场景和机器人。
- RViz 中 RobotModel、TF、LaserScan。

## 2. 目标二：启动导航功能

题目原意：

```text
启动 ROS2 Nav2 相关节点，比如地图、定位、规划器、控制器、行为树等。
```

对应实践文档：

- `Turtlebot3仿真导航快速跑通.md`
- `SLAM边建图边导航.md`
- `传感器建图定位与代价地图.md`
- `Nav2规划器与控制器插件配置.md`

### 2.1 已知地图导航

如果题目给了地图，通常走：

```text
map_server + AMCL + Nav2
```

Turtlebot3 示例：

```bash
cd ~/ros2_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch tb3_nav_practice turtlebot3_nav2_local.launch.py
```

RViz 操作：

1. 点击 `2D Pose Estimate`。
2. 按 Gazebo 里的真实位置设置机器人初始位姿。
3. 观察红色雷达线是否和地图墙体重合。

### 2.2 未知地图导航

如果题目没有地图，只给 Gazebo 场景，走：

```text
slam_toolbox + Nav2
```

终端 2：

```bash
source /opt/ros/humble/setup.bash
ros2 launch slam_toolbox online_async_launch.py use_sim_time:=True
```

终端 3：

```bash
source /opt/ros/humble/setup.bash
ros2 launch nav2_bringup navigation_launch.py use_sim_time:=True
```

### 2.3 如果要挂载自定义参数文件

```bash
cd ~/ros2_ws
source install/setup.bash
ros2 launch nav2_bringup navigation_launch.py use_sim_time:=True params_file:=/path/to/nav2_params.yaml
```

如果是包内参数：

```bash
CONFIG_DIR=$(ros2 pkg prefix <package_name>)/share/<package_name>/config/nav2_params.yaml
ros2 launch nav2_bringup navigation_launch.py use_sim_time:=True params_file:=$CONFIG_DIR
```

### 2.4 这一阶段要检查什么

```bash
ros2 node list
ros2 lifecycle nodes
ros2 topic echo /map --once
ros2 run tf2_ros tf2_echo map odom
```

重点节点：

- `planner_server`
- `controller_server`
- `bt_navigator`
- `behavior_server`
- `map_server` 或 `slam_toolbox`
- `amcl`，如果是已知地图导航
- `global_costmap`
- `local_costmap`

截图建议：

- RViz 中地图、机器人、雷达、代价地图。
- Nav2 已启动后 RViz 的完整界面。

## 3. 目标三：调整导航参数，让路径规划顺畅

题目原意：

```text
改参数，让机器人不要乱绕、卡住、撞障碍、贴墙、转圈或规划失败。
```

对应实践文档：

- `Nav2代价地图参数配置.md`
- `机器人Footprint轮廓配置.md`
- `Nav2规划器与控制器插件配置.md`
- `Nav2调参与故障诊断手册.md`

核心文件通常是：

```text
config/nav2_params.yaml
```

### 3.1 先改 Costmap

常见参数：

```yaml
global_costmap:
  global_costmap:
    ros__parameters:
      robot_radius: 0.22
      inflation_layer:
        inflation_radius: 0.55
        cost_scaling_factor: 3.0

local_costmap:
  local_costmap:
    ros__parameters:
      width: 3
      height: 3
      inflation_layer:
        inflation_radius: 0.55
        cost_scaling_factor: 3.0
```

现象对应：

- 路径贴墙：增大 `inflation_radius` 或减小 `cost_scaling_factor`。
- 不敢过窄门：减小 `inflation_radius`，检查机器人尺寸。
- 动态障碍清不掉：检查 `clearing`、`raytrace_max_range`。

### 3.2 再改 Footprint

如果机器人不是圆形，推荐配置 `footprint`：

```yaml
local_costmap:
  local_costmap:
    ros__parameters:
      footprint: "[ [0.21, 0.195], [0.21, -0.195], [-0.21, -0.195], [-0.21, 0.195] ]"
```

现象对应：

- 机器人蹭墙：footprint 可能太小。
- 机器人过不了明明能过的通道：footprint 可能太大。

RViz 检查：

```text
/local_costmap/published_footprint
/global_costmap/published_footprint
```

### 3.3 再改 Planner

常见配置：

```yaml
planner_server:
  ros__parameters:
    planner_plugins: ["GridBased"]
    GridBased:
      plugin: "nav2_smac_planner/SmacPlanner2D"
      tolerance: 0.125
      allow_unknown: true
```

现象对应：

- 不生成路径：看 planner 和 global costmap。
- 未知区域不规划：看 `allow_unknown`。
- 目标附近规划失败：适当放宽 `tolerance`。

### 3.4 最后改 Controller

常见配置：

```yaml
controller_server:
  ros__parameters:
    controller_frequency: 20.0
    controller_plugins: ["FollowPath"]
    FollowPath:
      plugin: "dwb_core::DWBLocalPlanner"
      max_vel_x: 0.26
      max_vel_theta: 1.0
```

现象对应：

- 有路径但不走：看 controller、TF、local costmap、`/cmd_vel`。
- 来回摆动：降低速度和角速度。
- 到终点附近转圈：看 goal checker 和目标角度容差。

### 3.5 修改后如何生效

通常需要重新启动 Nav2：

```bash
Ctrl+C 关闭 Nav2
source install/setup.bash
ros2 launch nav2_bringup navigation_launch.py use_sim_time:=True params_file:=/path/to/nav2_params.yaml
```

如果改的是包内文件，可能需要重新编译：

```bash
cd ~/ros2_ws
colcon build --packages-select <package_name>
source install/setup.bash
```

截图建议：

- 截 `nav2_params.yaml` 中改过的参数。
- 截 RViz 中 costmap、footprint、规划路径。

## 4. 目标四：启动导航，让机器人抵达终点

题目原意：

```text
机器人能在仿真场景中自动规划路径，并到达题目指定目标点。
```

对应实践文档：

- `Turtlebot3仿真导航快速跑通.md`
- `SLAM边建图边导航.md`
- `Nav2调参与故障诊断手册.md`
- `实践提交与截图清单.md`

### 4.1 RViz 下发目标

已知地图导航：

1. 先点 `2D Pose Estimate`。
2. 等定位收敛。
3. 点击 `Nav2 Goal`。
4. 在地图上拖出目标点和朝向。

SLAM 导航：

1. 确认 `/map` 已经在 RViz 中出现。
2. 点击 `Nav2 Goal`。
3. 在可通行区域设置目标点。

### 4.2 观察是否成功

RViz 中应看到：

- 生成全局路径。
- 局部代价地图围绕机器人滚动。
- 机器人沿路径移动。
- 到达终点后停止。

Gazebo 中应看到：

- 机器人真实移动。
- 没有撞墙。
- 没有长时间卡住或转圈。

### 4.3 如果失败，按这个顺序排查

1. `/scan` 有没有。
2. `/odom` 有没有。
3. `map -> odom` 有没有。
4. `odom -> base_link` 有没有。
5. Nav2 生命周期节点是否 active。
6. global costmap 是否显示。
7. local costmap 是否显示。
8. 是否有 `/cmd_vel`。
9. 目标点是否落在障碍物或未知区。
10. 再调 costmap、footprint、planner、controller。

对应文档：

```text
Nav2调参与故障诊断手册.md
```

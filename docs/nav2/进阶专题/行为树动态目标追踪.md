# 动态目标追踪：用行为树把固定终点变成移动目标

## 本章在导航系统中的位置

普通导航的目标点是一次性给定的。动态追踪的核心不是重写 planner/controller，而是让行为树不断接收新的目标，并触发重新规划。

本章对应链路：

```text
动态目标话题
  -> 行为树 GoalUpdater
  -> 黑板 updated_goal
  -> ComputePathToPose 重新规划
  -> FollowPath 持续跟随
```

这属于进阶能力。模块 C 普通“到指定终点”不需要先学它，但如果题目要求跟随人、目标点刷新、动态追踪，就回到这篇。

# 基于行为树 (Behavior Tree) 的动态目标跟随完整操作指南

## 1. 核心理论：行为树驱动的逻辑重构

在传统的导航任务中，机器人的目标是静态的（从 A 点移动到 B 点）。但在诸如“行李跟随”或“人员动态追踪”等高级自主系统中，目标物体是持续移动的。

本指南的核心思想是：**保持底层的规划器和控制器算法不变，仅通过重构 Nav2 的行为树（Behavior Tree, BT）逻辑，即可实现无限期的动态目标追踪。**

在真实的物理工程中，动态目标的空间坐标通常由前段的视觉算法（如深度学习目标检测模型）实时解算并发布。为了专注于验证导航行为控制逻辑，本节的仿真实验将利用 RViz 界面中的鼠标点击功能（`Clicked Point`）来模拟视觉检测器不断刷新的目标坐标。

## 2. 工程实现一：逐步重构追踪行为树

行为树是以 XML 格式编写的逻辑控制流。我们将从最基础的导航树开始，分四个阶段为其注入“动态追踪”的基因，最终形成一份完整的自定义行为树文件。

- **阶段一：基础路径规划与跟随。** 原始逻辑基于 `PipelineSequence` 管道序列：以 1Hz 的频率计算通往目标的路径，并将路径交给控制器执行。
- **阶段二：注入“无限续航”机制 (`KeepRunningUntilFailure`)。** 默认的 `FollowPath` 节点在到达目标后会返回 `SUCCESS` 并终止。我们在其外层包裹 `KeepRunningUntilFailure` 节点，使机器人无视“到达”状态，永远保持追踪激活。
- **阶段三：注入“目标动态刷新”机制 (`GoalUpdater`)。** 引入该节点持续监听 `/goal_update` 话题。一旦接收到新坐标，它会将内部的 `goal` 变量瞬间替换为 `updated_goal`，驱使规划器立刻朝新坐标重新布线。
- **阶段四：注入“安全社交距离” (`TruncatePath`)。** 为防止机器人直接撞上被追踪的动态目标，引入截断节点。该节点会自动截断生成的路径，使机器人在距离目标特定距离（例如 `distance="1.0"` 米）处提前刹停。

### 编写终极行为树文件

请在终端中执行以下命令，创建行为树专属目录及文件：

Bash

```
mkdir -p ~/ros2_ws/src/sam_bot_description/behavior_trees
gedit ~/ros2_ws/src/sam_bot_description/behavior_trees/follow_dynamic_object.xml
```

将整合上述四大逻辑的完整 XML 代码填入并保存：

XML

```
<root main_tree_to_execute="MainTree">
  <BehaviorTree ID="MainTree">
    <PipelineSequence name="NavigateWithReplanning">
      <RateController hz="1.0">
        <Sequence>
          <GoalUpdater input_goal="{goal}" output_goal="{updated_goal}">
            <ComputePathToPose goal="{updated_goal}" path="{path}" planner_id="GridBased"/>
          </GoalUpdater>
          <TruncatePath distance="1.0" input_path="{path}" output_path="{truncated_path}"/>
        </Sequence>
      </RateController>
      <KeepRunningUntilFailure>
        <FollowPath path="{truncated_path}" controller_id="FollowPath"/>
      </KeepRunningUntilFailure>
    </PipelineSequence>
  </BehaviorTree>
</root>
```

## 3. 工程实现二：规范化集成与编译

## 4. 仿真联调与追踪验证

代码集成完毕后，我们将开启三个独立的终端，进行全链路的动态追踪验证。

### 步骤 1：部署视觉信号模拟器

打开**第一个终端**，运行 ROS 2 官方提供的轻量级转换节点。该节点负责将 RViz 中鼠标点击产生的 `/clicked_point` 话题，转换为行为树能够识别的目标更新话题（`/goal_update`）。

Bash

```
source /opt/ros/humble/setup.bash
ros2 run nav2_test_utils clicked_point_to_pose
```

### 步骤 2：加载自定义行为树启动 Nav2

打开**第二个终端**，使用标准动态路径获取指令，将默认导航树替换为我们自定义的追踪树，并启动仿真：

Bash

```
source ~/ros2_ws/install/setup.bash
export TURTLEBOT3_MODEL=waffle

# 动态获取自定义行为树的绝对路径
BT_DIR=$(ros2 pkg prefix sam_bot_description)/share/sam_bot_description/behavior_trees/follow_dynamic_object.xml

# 启动仿真导航，并强行注入追踪行为树
ros2 launch nav2_bringup tb3_simulation_launch.py default_bt_xml_filename:=$BT_DIR
```

### 步骤 3：交互式动态追踪测试

系统启动并弹出 RViz2 监控界面后：

1. 观察 RViz 界面，确认机器人已在地图中完成 AMCL 初始定位（雷达红线与地图墙壁重合）。
2. 在 RViz 顶部工具栏中选择 **`Publish Point` (点击点)** 按钮。
3. 在机器人前方的空地上点击任意一点，机器人将立刻规划出绿色路线并开始移动。
4. **动态验证（核心操作）：** 在机器人移动的过程中，不要等它停下，继续使用 `Publish Point` 工具在地图不同位置**连续快速点击**（这模拟了视觉检测器以高频率不断输出行人移动的新坐标）。
5. **预期现象观察：** * 机器人不会出现卡顿或停滞，而是实时且流畅地改变行驶轨迹，如同猎犬般持续追逐最新点击的坐标。
   - 当停止点击且机器人逼近最后一次点击的位置时，它不会直接驶过该点，而是会在距离该点正好 1.0 米的半径边界处精准刹停，完美维持预设的安全社交距离。

知识点



| **节点名称 (XML 标签)**         | **节点类型** | **核心工程功能**                                             | **黑板变量 (输入 → 输出)**              |
| ------------------------------- | ------------ | ------------------------------------------------------------ | --------------------------------------- |
| **`<PipelineSequence>`**        | 控制节点     | **并发调度。** 依次启动子节点。当子节点 1（规划）返回成功时，启动子节点 2（控制），随后**同时并行**执行子节点 1 和 2。 | 无                                      |
| **`<RateController>`**          | 装饰节点     | **频率限制。** 强制其包裹的子树的执行频率不超过指定值（此处 `hz="1.0"`，即最高每秒触发 1 次），用于控制重规划造成的 CPU 负载。 | 无                                      |
| **`<Sequence>`**                | 控制节点     | **顺序执行。** 严格按照从上到下的顺序执行子节点。上一个节点返回 SUCCESS 后才执行下一个；若任一节点返回 FAILURE，则立即中断并返回 FAILURE。 | 无                                      |
| **`<GoalUpdater>`**             | 行为节点     | **动态目标更新。** 持续订阅并监听特定的 ROS 话题（默认 `/goal_update`）。当接收到新的位姿数据时，更新目标变量。 | 输入: `{goal}` 输出: `{updated_goal}`   |
| **`<ComputePathToPose>`**       | 行为节点     | **全局路径规划。** 调用 Nav2 的全局规划器（此处指定 `planner_id="GridBased"`），计算从机器人当前位置到目标点的二维全局路径。 | 输入: `{updated_goal}` 输出: `{path}`   |
| **`<TruncatePath>`**            | 行为节点     | **路径截断处理。** 根据指定的物理距离（此处 `distance="1.0"` 米），裁剪掉输入路径末尾的线段，生成一条缩短后的新路径。用于避开物理碰撞。 | 输入: `{path}` 输出: `{truncated_path}` |
| **`<KeepRunningUntilFailure>`** | 装饰节点     | **状态重写（拦截成功信号）。** 拦截子节点返回的 SUCCESS 状态，将其强行转换为 RUNNING 状态。使得动作不会因“到达终点”而停止，除非发生致命 FAILURE。 | 无                                      |
| **`<FollowPath>`**              | 行为节点     | **局部轨迹控制。** 调用 Nav2 的局部控制器（此处指定 `controller_id="FollowPath"`），消化输入的路径，并向硬件输出真实的底盘速度指令 (`cmd_vel`)。 | 输入: `{truncated_path}`                |

1.
